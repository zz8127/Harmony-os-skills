# 应用分发

> **适用版本**：HarmonyOS 6.1 / API 23（稳定）。兼容 API 14+。

## 概述

AppGallery Connect 应用分发是华为应用市场（AppGallery）的开发者端服务，覆盖应用从创建到上架的全流程。支持发布 HarmonyOS 应用/元服务、Android 应用、快应用、App Bundle 等多种类型包，覆盖 200+ 国家与地区，支持 78 种语言。

**官方文档**：https://developer.huawei.com/consumer/cn/doc/app/agc-help-app-0000002235710234

---

## 分发流程

```
创建应用 → 填写应用信息 → 上传软件包 → 提交审核 → 审核通过 → 上架发布
                                                    ↓
                                              审核驳回 → 修改后重新提交
```

---

## 创建应用

### 前提条件

1. 注册华为开发者账号并完成实名认证
2. 登录 [AGC 控制台](https://developer.huawei.com/consumer/cn/service/jsp/agc/index.html)

### 操作步骤

1. AGC 控制台 → 我的项目 → 添加应用
2. 选择平台：HarmonyOS / Android
3. 填写应用名称、包名（bundleName）
4. 下载 `agconnect-services.json` 配置文件

> ⚠️ 包名创建后**不可修改**，请务必确认。

## 应用信息

| 信息项 | 说明 | 要求 |
|--------|------|------|
| 应用名称 | 展示在 AppGallery 的名称 | 2-64 字符 |
| 应用图标 | 应用展示图标 | 512×512 PNG |
| 应用截图 | 功能截图 | 至少 3 张，最大 10 张 |
| 应用简介 | 简短说明 | 4-200 字符 |
| 应用描述 | 详细介绍 | 8-16000 字符 |
| 应用类别 | 分类归属 | 必选一级+二级分类 |
| 隐私政策 | 隐私声明 URL | 必填 |
| 年龄分级 | 适用年龄 | IARC 分级 |

## 版本管理

### 版本类型

| 类型 | 说明 |
|------|------|
| 正式版 | 面向所有用户发布 |
| 灰度版 | 分阶段逐步放量 |
| 测试版 | 仅测试用户可见 |

### 软件包要求

| 项目 | 要求 |
|------|------|
| 格式 | .app（HarmonyOS）/ .apk / .aab（Android） |
| 签名 | 必须使用发布证书签名 |
| 大小 | App Pack ≤ 4GB |
| 版本号 | versionCode 递增 |

### 上传方式

1. **控制台手动上传**：AGC 控制台 → 版本信息 → 上传软件包
2. **DevEco Studio 上传**：Build → Build Hap(s)/APP(s) → Build APP(s) → 上传
3. **API 上传**：通过 Publishing API 自动化上传

## 审核规范

### 审核流程

1. 提交审核 → 2. 自动安全扫描 → 3. 人工审核 → 4. 审核结果

### 审核时间

| 审核类型 | 时间 |
|---------|------|
| 首次提交 | 1-3 个工作日 |
| 更新提交 | 1-2 个工作日 |
| 加急审核 | 需申请 |

### 常见审核驳回原因

| 原因 | 说明 |
|------|------|
| 功能异常 | 崩溃/闪退/核心功能不可用 |
| 隐私合规 | 未声明权限用途/过度收集信息 |
| 内容违规 | 色情/暴力/赌博/虚假宣传 |
| 侵权问题 | 侵犯知识产权/品牌冒用 |
| UI 不规范 | 关键操作无反馈/适配问题 |
| 签名问题 | 调试签名/签名不一致 |

### 鸿蒙应用审核注意事项

- 必须基于 Stage 模型开发（FA 模型不支持 HarmonyOS NEXT）
- 应用需支持星盾安全访问机制
- 不得使用未声明的系统权限
- 隐私政策必须包含数据收集说明

## 发布管理

### 发布方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| 全量发布 | 所有用户同时收到更新 | 常规更新 |
| 灰度发布 | 按比例逐步放量 | 大版本/风险较高 |
| 分阶段发布 | 按时间分阶段放量 | 关键业务应用 |

### 灰度发布配置

1. AGC 控制台 → 版本信息 → 发布方式 → 选择灰度
2. 设置灰度比例：5% → 10% → 25% → 50% → 100%
3. 监控指标：崩溃率、ANR率、用户反馈
4. 确认无问题后提升比例，有问题则暂停

### 多渠道发布

- 支持同时发布到多个国家/地区
- 可按地区设置不同的上架时间
- 可配置不同的应用信息（多语言）

## 应用下架与更新

| 操作 | 说明 |
|------|------|
| 下架 | 从 AppGallery 移除，已安装用户仍可使用 |
| 取消发布 | 撤回审核中的版本 |
| 替换版本 | 上传新包覆盖未发布版本 |
| 紧急回滚 | 发现严重问题时回退到上一版本 |
