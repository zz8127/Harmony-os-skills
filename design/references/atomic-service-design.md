# 元服务设计规范

> **适用版本**：HarmonyOS 设计指南
> **所属分类**：设计规范 / 元服务

---

## 概述

在万物互联时代，人均持有设备量不断攀升，设备和场景的多样性使服务开发变得更加复杂、服务入口更加丰富。HarmonyOS 提供了基于元服务的便捷服务，使应用开发更简单，服务的获取和使用更便捷。

元服务为开发者提供了一个在 HarmonyOS 内搭建和实现特定服务、功能的平台，集成 HarmonyOS 系统级能力，将先进技术、连接触达、优质服务赋予开发者。

---

## 元服务核心特征

| 特征 | 说明 |
|------|------|
| 免安装 | 无需安装即可使用，降低用户使用门槛 |
| 轻量快捷 | 功能聚焦，架构简单轻量 |
| 随处可及 | 通过多种系统入口快速触达 |
| 即用即走 | 用完即走，无需长期驻留 |
| 一步直达 | 减少操作步骤，直达服务核心 |

---

## 元服务入口

用户可通过以下系统入口快速访问和获取元服务：

| 入口 | 说明 |
|------|------|
| 负一屏 | 系统桌面左滑进入，展示服务卡片 |
| 全局搜索 | 系统搜索栏直接搜索元服务 |
| 应用市场 | 在应用市场中浏览和发现元服务 |
| 小艺建议 | 系统智能推荐场景化服务 |
| 桌面 | 服务卡片添加到桌面 |
| 应用内调用 | 应用内可调用打开元服务 |
| HarmonyOS Connect 标签 | 扫描标签快速启动 |
| 碰一碰 | NFC 碰一碰设备快速启动 |

---

## 元服务适用场景

- 功能相对单一、架构简单轻量、用完即走的应用，可直接提供对应元服务
- 无需花费大量成本开发完整应用
- 需要多入口快速触达用户的服务
- 与硬件设备联动（如碰一碰）的场景化服务

---

## 元服务分发方式

HarmonyOS 为元服务提供了丰富多样的分发方式，服务可通过多种系统级入口快捷触达用户，提升服务的使用率，带来更多收益。

---

## 元服务基本要素

元服务的设计需要遵循 HarmonyOS "用户体验为先，一切皆服务，万物可分享"的价值观，共同打造多元化、连续化、智慧化的数字生态。
