# 多设备设计

> **适用版本**：HarmonyOS 6 / HarmonyOS NEXT

## 设备类型

| 设备类型 | 屏幕特点 | 主要交互方式 |
|---------|---------|------------|
| 手机 | 4-7 寸，触摸 | 手指滑动、点击 |
| 折叠屏 | 展开/折叠态 | 折叠切换、触摸 |
| 平板 | 8-13 寸，触摸 | 手指/手写笔 |
| 二合一设备 | 可变形态 | 触摸 + 键鼠 |
| 智慧屏 | 大屏远距离 | 遥控器、语音 |
| 智能手表 | 小屏手腕距离 | 手势、语音 |
| 车机 | 驾车场景 | 方向盘/语音/触摸 |

---

## 设计三原则（多设备通用）

| 原则 | 说明 |
|------|------|
| 差异性 | 针对设备特性针对性设计 |
| 一致性 | 跨设备体验保持连续性 |
| 协同性 | 多设备无缝流转体验 |

---

## 手机设计要点

| 要点 | 说明 |
|------|------|
| 触控优先 | 触控区域不低于 44×44 dp |
| 单手操作 | 常用操作区域靠近屏幕底部 |
| 信息密度 | 适中小屏信息量 |

---

## 折叠屏设计要点

| 要点 | 说明 |
|------|------|
| 内容完整 | 展开态信息量不少于折叠态的 3/4 |
| 字号稳定 | 展开态字号不超过折叠态的 1.2 倍 |
| 弹出框 | 展开态不超过折叠态的 1.2 倍 |
| 屏幕旋转 | 旋转后布局平滑过渡 |

---

## 平板 / 二合一设计要点

| 要点 | 说明 |
|------|------|
| 多窗口 | 支持分屏/悬浮窗口 |
| 键鼠适配 | 悬停状态、快捷键、右键菜单 |
| 平板笔 | 支持手写笔压感 |
| API 14+ | 支持自定义应用启动页 |

---

## 智慧屏设计要点

| 要点 | 说明 |
|------|------|
| 远距离可读 | 字号比手机大 1.5-2 倍 |
| 遥控器交互 | 焦点导航清晰，方向键导航顺畅 |
| 语音交互 | 支持远场语音指令 |
| 暗光环境 | 深色模式为主 |

### 智慧屏 UX 核心指标

| 指标 | 要求 |
|------|------|
| 阅读效率 | 用户 2 秒内完成主要信息阅读 |
| 触控距离 | 避免控件距离太远难以触及 |
| 安全区域 | 避免方向盘遮挡关键控件 |

---

## 智能手表设计要点

| 要点 | 说明 |
|------|------|
| 信息精简 | 只展示最关键信息 |
| 字号更大 | 确保手腕距离可读 |
| 粗体字重 | 使用粗体增强可辨性 |
| 表盘适配 | 确保在不同表盘形状上显示正常 |

---

## 车机设计要点

| 要点 | 说明 |
|------|------|
| 驾驶安全 | 减少驾驶员分心，关键信息快速获取 |
| 触控优化 | 控件满足最小触控尺寸要求 |
| 语音优先 | 复杂操作优先使用语音指令 |
| 暗色模式 | 暗光环境为主的界面设计 |

---

## 多设备协同（分布式设计）

HarmonyOS 的核心优势在于多设备协同：

| 协同模式 | 说明 |
|---------|------|
| 设备互联 | 手机与平板/智慧屏快速连接 |
| 能力互助 | 调用其他设备的硬件能力（如用平板调用手机摄像头） |
| 资源共享 | 跨设备数据无缝同步 |
| 任务流转 | 应用在设备间自由迁移 |

---

## 响应式适配流程

```
1. 定义设备差异性 → 2. 提取共性设计 → 3. 构建响应式布局
```

| 步骤 | 内容 |
|------|------|
| 差异性分析 | 确定目标设备的屏幕、交互、场景特点 |
| 共性提取 | 找出跨设备统一的设计模式 |
| 适配实现 | 使用栅格系统、相对单位、弹性布局实现响应式 |
