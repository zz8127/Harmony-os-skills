# 针对多设备设计

> **适用版本**：HarmonyOS NEXT 设计指南。兼容 HarmonyOS 6.0+。

## 概述

HarmonyOS 支持多种设备类型，包括手机、平板、折叠屏、电脑、智慧屏、智能穿戴等。针对多设备设计需要充分考虑不同设备的硬件特性、使用方式、交互方式和使用场景，确保应用在各设备上都能提供优秀的用户体验。

---

## 手机设计规范

### 硬件特性

| 特性 | 说明 |
|------|------|
| 屏幕 | 高分辨率的中等大小屏幕 |
| 摄像头 | 配备优质后置和前置摄像头 |
| 音频 | 较好的音频输入输出能力 |
| 生物认证 | 一般配备指纹、人脸识别 |
| 地理位置 | 可通过 GPS 获取地理位置 |
| 其他传感器 | 陀螺仪、加速计等运动状态传感器 |

### 使用方式

- 用户随身携带，随时使用
- 根据习惯和场景单手或双手握持
- 主要竖屏使用，需要时切换横屏
- 使用距离通常约 30 厘米

### 交互方式

- 主要交互方式：触屏手势
- 语音交互：随着 AI 能力增强越来越普遍
- 多设备协同时：鼠标拖拽交互

### 设计要点

| 要点 | 说明 |
|------|------|
| 使用系统控件 | 利用底部页签、标题栏、弹出框等标准控件 |
| 使用合适的应用架构 | 内容类应用通常采用底部页签架构 |
| 保证页面正常显示 | 支持横屏显示，针对挖孔进行适配 |
| 考虑单手操作 | 核心功能放在单手容易触达的区域 |
| 避免多指手势 | 尽量不使用两指以上的多指手势 |
| 充分利用硬件特性 | 人脸识别、指纹、陀螺仪等减少用户操作负担 |

## 系统特性接入

| 特性 | 说明 |
|------|------|
| 导航条 | 系统导航交互 |
| 通知 | 系统通知展示 |
| 实况窗 | 实时状态展示（如外卖配送状态） |
| 多窗口交互 | 多窗口协同操作 |
| 画中画 | 小窗视频播放 |
| 深色模式 | 跟随系统深色模式切换 |
| 状态栏 | 系统状态栏适配 |
| 播控中心 | 媒体播放控制 |

### 接入原则

1. **充分利用系统特性**：根据业务属性接入系统特性，获得更多触达用户的机会
2. **遵循系统特性的体验要求**：如接入实况窗需按模板设计布局和信息
3. **跟随系统设置**：如深色模式应跟随系统实时切换

## 多设备类型概览

| 设备类型 | 关键设计考量 |
|----------|-------------|
| 手机 | 单手操作、竖屏优先、触屏交互 |
| 平板 | 多窗口、手写笔、分屏布局 |
| 折叠屏 | 开合态切换、悬停态适配 |
| 双折叠 | 屏幕高度图文优化 |
| 三折叠 | 大屏适配与多任务 |
| 阔折叠 | 翻盖折叠态设计 |
| 电脑 | 窗口框架、键鼠交互、应用状态栏接入 |
| 智慧屏 | 远距离交互、遥控器操作 |
| 智能穿戴 | 圆形/方形表盘、小屏精简设计 |
