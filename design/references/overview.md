# 设计原则概述

> **适用版本**：HarmonyOS 6.1 / API 23（稳定）；HarmonyOS 6.0 / API 22（稳定）。兼容 API 14+。

## HarmonyOS Design 设计哲学

### 简洁高级

HarmonyOS Design 追求**简洁高级**的设计风格，核心体现在：

| 特征 | 说明 |
|------|------|
| 简洁清晰 | 减少视觉噪音，核心内容和关键操作突出展示 |
| 界面一致 | 相同场景保持统一布局、颜色、字体、材质 |
| 内容易读 | 图文清晰，标准字号色彩方案，满足暗光/暗色场景 |
| 多彩有序 | 色彩和谐，避免杂乱无章 |

### 平衡性原则

| 类型 | 说明 |
|------|------|
| 元素图形平衡 | 视觉重量均匀分布 |
| 元素间大小重量平衡 | 同级元素视觉权重相当 |
| 页面视觉平衡 | 整体画面重心稳定 |
| 排布对齐 | 元素排布整齐，文本排版对齐 |

### 层级性原则

| 层级类型 | 实现手段 |
|---------|---------|
| 深度层级 | 色彩明度、阴影、模糊构造深度线索 |
| 平面层级 | 字号差异、色彩饱和度差异、元素大小差异构造平面层级 |

### 信息密度控制

- 界面信息种类数适中（大小、色彩数量等）
- 界面信息数适中（组块数量、文字数量等）
- 适当留白，关键操作控件足够明显
- 关键信息呈现在核心视域

## 设计分层体系

### 四层架构

```
战略层 → 范围层 → 框架层 → 表现层
```

| 层级 | 职责 |
|------|------|
| 战略层 | 定义用户体验目标（流畅性、无障碍等） |
| 范围层 | 明确功能边界与交互模式（如分布式操作规范） |
| 框架层 | 提供标准化组件库（Button、List 等 80+ 组件） |
| 表现层 | 视觉规范（色彩系统、动效曲线、字体规范） |

---

## 原子化设计

HarmonyOS 采用原子化设计理念：

- **Atom（原子组件）**：基础组件，如 Button、Icon
- **Molecule（分子组件）**：原子组件组合，如 IconButton = Button + Icon
- **Organism（有机组件）**：复杂组件组合

通过基础原子组件组合生成复杂组件，提高复用性。

---

## 设计目标

1. **一致性的交互定义**：遵循统一交互逻辑，提高产品易学易用性
2. **标准化设计**：一次开发满足所有设备适配，简单高效
3. **内容易读性**：图文清晰，满足不同光照场景显示效果
4. **控制任务内容信息量**：使用户能在2秒内完成阅读
5. **关键信息位置**：呈现在用户核心视域，避免被遮挡

---

## 设计一致性要求

- 每个界面相同场景处理方式保持一致
- 包括：布局、颜色、材质、字体
- 功能的可用/不可用状态需明确区分并清晰呈现
