# 色彩系统

> **适用版本**：HarmonyOS 6 / HarmonyOS NEXT

## 主色调：宇宙蓝

HarmonyOS 采用**蓝色**作为系统主色调。

**选择蓝色的原因**：
- 人因研究：蓝色在男性/女性群体中接受度最高
- 全球地域维度：蓝色是最受欢迎的颜色
- 无障碍友好：大多数色觉障碍人士仍可辨识蓝色

我们选择更为**深邃广阔**的宇宙蓝，无边无垠，深邃如宇宙，为系统带来宇宙初开的纯净与宁静。

| 名称 | 色值 | 使用场景 |
|------|------|---------|
| 宇宙蓝（Primary） | #007DFF | 主色调、关键操作、重点信息 |

## Token 分层机制

HarmonyOS Design 采用**Token 参数**机制，实现色彩与实现解耦：

| 分层 | 说明 |
|------|------|
| 全局级 Token | 最基础的色彩变量（如 brand.color.primary） |
| 语义级 Token | 语义化命名（如 button.background.primary） |
| 平台级 Token | 具体平台适配值（如 button.bg.blue） |

**优势**：
- 设计规范与代码实现解耦
- 支持浅色/深色模式自动切换
- 全局修改一处，所有引用自动更新

## 浅色模式 vs 深色模式

### 深色模式适配要点

- 深色模式下文字对比度需符合无障碍标准
- 状态栏颜色需跟随深色模式切换
- 卡片背景在深色模式下避免纯黑（#000000），建议使用深灰
- 避免硬编码颜色，统一使用 Token 变量

### 深色模式建议

| 元素 | 浅色模式 | 深色模式 |
|------|---------|---------|
| 背景 | #FFFFFF | #000000 / #1C1C1E |
| 卡片 | #F5F5F5 | #2C2C2E |
| 主文字 | #000000 | #FFFFFF |
| 次文字 | #666666 | #8E8E93 |

## 应用色彩规范

| 原则 | 说明 |
|------|------|
| 风格协调性 | 应用整体色彩风格统一 |
| 图标可识别性 | 图标/文字颜色与背景有足够对比度 |
| 色彩合理性 | 配色和谐，不引起视觉不适 |
| 避免同质化 | 避免大量使用纯色或渐变 |
