## 应用设计最佳实践

### 概述

随着终端设备形态日益多样化，应用设计需要考虑界面适配不同屏幕尺寸、屏幕方向和设备类型。同时需要保持多设备体验的连续性，改善多端独立设计，降低开发者的工作量和维护成本。本指南结合用户在多端设备上的历史交互习惯与使用诉求，提供面向多设备的设计方法。

### 四大设计模块

| 模块 | 说明 |
|------|------|
| 基础要求 | 多设备应用设计中必须遵守的基础体验要求 |
| 响应式布局 | 针对常见界面元素提出宽屏设备上的响应式布局设计范式 |
| 增值体验 | 多设备应用设计中可考虑的更多体验变化 |
| 场景化设计参考 | 针对具体垂类场景应用给出的设计建议 |

## 基础要求

### 导航适配

- 手机和折叠屏使用底部页签导航
- 平板及宽屏设备使用侧边页签或侧边栏导航
- 调用鸿蒙系统控件可自动适配导航规则
- 应用内底部元素需自动抬高避让底部导航条
- 沉浸式页面超过 2 秒自动隐藏导航条

### 横竖屏 & 挖孔适配

- 手机和折叠屏折叠态适配竖屏布局
- 折叠屏展开态、平板需同时适配横屏和竖屏布局
- 设备尺寸比例接近 1:1 时，横竖屏使用相同或相近布局
- 核心内容或重要交互不要被挖孔区遮挡
- 避免因避让挖孔导致左右侧留白不一致

### 多窗适配

- 手机和折叠屏折叠态：适配竖向悬浮窗、上下分屏
- 折叠屏展开态：额外适配上下分屏
- 平板及更宽设备：适配自由窗口、左右分屏、分屏比例自由调节
- 长视频、直播、会议、通话等场景需支持画中画

### 弹出框适配

- 建议调用系统弹出框控件
- 自定义弹出框在 8 栅格及以上设备上最宽不超过 400vp
- 宽屏设备弹出框物理高度不超过手机上该弹出框高度的 1.5 倍

### 键鼠适配

- 平板及更宽设备上的应用需进行基础鼠标、键盘适配

## 响应式布局

### 布局策略

| 策略 | 说明 |
|------|------|
| 形变 | 根据屏幕宽度调整元素比例 |
| 延伸布局 | 横向延伸内容区域 |
| 重复布局 | 宽屏显示多列重复内容 |
| 挪移布局 | 将内容从一列挪移到多列 |
| 宫格布局 | 从列表变为宫格展示 |
| 瀑布流布局 | 从列表变为瀑布流展示 |

### 具体适配要点

- **子页签 & 搜索**：手机分两排显示，宽屏设备同一排显示
- **入口图标**：折叠屏/平板竖屏一排不超过 8 个，平板横屏不超过 13 个
- **广告图**：卡片广告使用延伸布局和形变适配；沉浸广告背景和内容分层
- **列表**：通过重复布局、挪移布局、变瀑布流/宫格改善浏览体验
- **卡片**：宽屏设备通过分栏 + 瀑布流/宫格布局显示
- **详情大图**：通过延伸布局或挪移布局适配
- **内容模块**：根据多个模块内容尺寸进行自适应挪移布局

## 增值体验

### 沉浸体验

| 场景 | 说明 |
|------|------|
| 沉浸广告 | 广告背景与内容分层，根据屏幕宽度自适应布局 |
| 沉浸浏览 | 上滑隐藏标题栏/工具栏，下滑或停留恢复显示 |
| 沉浸观影 | 减少屏幕旋转；减少弹幕对视频内容的遮挡 |

### 高效体验

| 场景 | 说明 |
|------|------|
| 分栏 | 8 栅格以上设备支持分栏，适用于办公/效率/IM 类应用 |
| 应用内分屏 | 通过"分屏"按钮触发任务分屏，支持文档对比、多任务协同 |
| 缩放 | 瀑布流/宫格支持双指缩放调整列数；图文阅读支持双指缩放调整字号 |

### 轻交互

| 场景 | 说明 |
|------|------|
| 浅层窗口 | 新建/筛选/添加/支付/登录等页面通过半模态控件避免大幅页面跳转 |
| 侧边面板 | 边看边评、边看边买等便捷任务处理 |
| 临时双窗 | 临时打开辅助窗口 |
| 长按预览 | 长按快速预览内容 |

### 跟手交互

- 跟手弹框：弹框跟随操作位置显示
- 跟手输入：输入框跟随焦点位置

### 折叠屏悬停

- 悬停适配：折叠屏半开状态下的布局适配
- 悬停创新：利用悬停状态提供创新交互

## 应用 UX 体验标准

### 概述

本标准从影响用户体验的各个维度定义了测试规范，规定了应用需达到的基础体验要求，用于引导应用的设计与开发，保证应用良好的使用体验。适用于应用的 UX 设计、开发与测试。

### 术语定义

| 术语 | 说明 |
|------|------|
| 系统手势 | 操作系统定义的触摸屏手势，用于完成系统级操作 |
| 点击热区 | 屏幕上可供用户点击/触摸/交互的区域 |
| 深色模式 | 适合黑暗环境使用的色彩模式，深色背景 + 浅色前景 |

### 单位定义

| 单位 | 说明 |
|------|------|
| vp | 虚拟像素单位（virtual pixel），适应不同屏幕密度 |
| fp | 字体像素单位（font pixel），1fp = 1vp × 缩放系数 |
| LTR/RTL | 阅读顺序：从左到右 / 从右到左 |

### 通用 UX 体验标准

涵盖应用交互的各维度基础体验要求，包括但不限于：
- 界面布局与导航
- 触控与手势交互
- 视觉反馈与动效
- 无障碍访问
- 多语言与国际化
