# 布局规范

> **适用版本**：HarmonyOS 6 / HarmonyOS NEXT

## 栅格系统

HarmonyOS 采用 **12 栅格**系统作为响应式布局的基础。

### 栅格参数

| 参数 | 值 |
|------|---|
| 列数 | 12 列 |
| 最小单位 | 4 vp / 8 vp |
| 间距（Gutter） | 常见值：8 / 12 / 16 / 24 vp |
| 安全边距 | 常见值：12 / 16 / 24 vp |

---

## 响应式布局

### 断点（Breakpoint）

| 断点 | 屏幕宽度 | 设备类型 |
|------|---------|---------|
| xs | < 320 vp | 小屏手机 |
| sm | 320-600 vp | 手机 |
| md | 600-840 vp | 大手机/小平板 |
| lg | 840-1200 vp | 平板 |
| xl | > 1200 vp | 大平板/二合一设备 |

### 布局模式

| 模式 | 说明 |
|------|------|
| 单列布局 | 简单内容线性排列 |
| 双列布局 | 并列信息分栏展示 |
| 网格布局 | 卡片式信息网格排列 |
| 自适应布局 | 根据屏幕宽度自动调整列数 |

---

## 间距规范

### 常用间距值

| 名称 | 间距值 | 使用场景 |
|------|-------|---------|
| xs | 4 vp | 紧凑元素内部间距 |
| sm | 8 vp | 相关元素之间 |
| md | 12 / 16 vp | 区块内部 |
| lg | 24 vp | 区块之间 |
| xl | 32 / 48 vp | 大区块之间 |
| xxl | 64 vp | 页面级留白 |

### 布局平衡原则

- 元素间间距应保持一致（同类元素使用相同间距）
- 区块之间使用较大间距，元素内部使用较小间距
- 关键操作区域的控件间距适当放大，避免误触

---

## 折叠屏适配

| 要求 | 说明 |
|------|------|
| 内容信息完整 | 展开态信息量不少于折叠态的 3/4 |
| 元素比例 | 图片/字体在展开态变化不超过 1.2 倍 |
| 弹出框适配 | 展开态与折叠态弹出框大小比例不超过 1.2 倍 |
| 屏幕旋转 | 旋转后布局需平滑过渡，避免内容被截断 |

---

## 布局设计原则

| 原则 | 说明 |
|------|------|
| 简洁清晰 | 布局结构明了，避免过于繁杂 |
| 信息突出 | 核心内容和关键操作优先展示 |
| 视觉平衡 | 元素分布均匀，重心稳定 |
| 留白合理 | 适当留白提升可读性，避免拥挤 |
